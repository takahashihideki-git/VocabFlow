# 位相同期の分散 — 播種ノイズ（seed noise）の探索

**位置づけ:** 研究ノート（検証結果の記録）。`bayesian-srs-proposal.md` §3（トンプソンサンプリングによる位相同期の分散）の検証を起点に、別解にたどり着いた経緯と結論。**播種ノイズは未だ core 未実装**（採否判断待ち）。検証は `scripts/verify_seed_noise.js`・`scripts/verify_due_sampling.js`。

---

## 1. 出発点 — dueSampling（提案書 §3）の再検証

提案書 §3 は、due 判定時に `effectiveH = h × (1 ± uncertaintyWidth)` をサンプリングして位相同期を散らす案（既定 `dueSampling: true`）。当初検証では「定着 +7%」とされたが、**それは旧 virtual-learner（`trueH = システムの h × 静的個体差`）下の値**で、提案書 §8 自身が「不適切」と警告したモデルだった。

review #1 の検証で virtual-learner を**間隔効果ありの独立した真の記憶モデル**に刷新後、**N=24** で再検証:

| 学習者 | Δ定着 | 判定 |
|---|---|---|
| 初学者（ability=1.0・hVariation=0.3） | +0.2（SE±1.7） | 有意でない |
| 既習（ability=1.5・hVariation=0.05） | +1.3（SE±2.2） | 有意でない |

throughput への効果は立証できず。需要 CV は再現的に ~0.02 低下する（有意・サンプリング本来の平滑化）が、**ピーク需要は不変**（朝バーストではむしろ +1.6 悪化）。当初の +7% は旧 learner のアーティファクトだった。

**教訓1（小標本ノイズ）**: throughput は run 間 std（±6〜9）が真値（±1〜2）より大きく、5回平均では符号すら定まらない。3回→5回→別5回で「効果なし→+5〜7%→−4〜−2」と二転三転し、N=24 でようやく「Δ≈0」に収束。**タイミング系機構の検証は N≥20 + 標準誤差必須**。

---

## 2. 再フレーム — 「h は seed であって測定値ではない」

ユーザー指摘: **h は真の記憶の測定値ではなく、SRS をなめらかに回すための制御変数（seed）**。外界に「真の h」があってそれを誤差付きで近似しているわけではない。だから「校正 MAE が悪化＝h が汚れた＝コスト」という判定は、手段それ自体の純度を測って目的と取り違えている。

決定的な点: **MAE はバイアスと分散を混ぜている**。
- **#1（deltaT-gain）が直したのは系統バイアス**: OFF は h を系統的に過大評価 → 復習が遅すぎる → 実際に忘れる → 本物の害。だから #1 の MAE 改善は**アウトカム改善（定着 131→334）を伴った**。
- **対称ノイズが足すのは分散**: 平均ゼロなので系統的な誤スケジュールを生まない。MAE が上がるのは注入したノイズが写るだけ＝同語反復で、害ではない。

→ 機構は**アウトカム（学習者が実際に覚えているか）**で判定すべき。h の純度では測らない。

---

## 3. 別解 — h 更新への信頼度連動ノイズ（保存型）

dueSampling（due 判定時・引き直し・h に保存しない）に対し、**h 更新そのものに信頼度連動ノイズを保存型で乗せる**。コホートの h が恒久的に分岐し、due 時刻がずれて位相同期が壊れる。

### 3.1 鍵は「勾配」

ノイズ幅 `w = base / rc^exp`（base=0.5・rc=更新後 reviewCount）。

| 勾配 | rc=1 | rc=2 | rc=4 | 成熟語(rc≥4)への影響 |
|---|---|---|---|---|
| `base/√rc`(floor 0.05) | 0.5 | 0.35 | 0.25 | ±5〜25% 残存 → **複利蓄積で h を汚す** |
| `base/rc^2.5` | 0.5 | 0.088 | 0.020 | rc=1 に集中・rc≥2 で実質ゼロ → **「導入時の一回播種」** |

固定 ±20%（劣化版）はグリッド量子化に丸められて死んだ。`√rc`（提案書のデフォルト幅）は緩すぎて成熟語まで侵食。**`rc^2.5` で播種を出生時に閉じ込めると複利が止まる**。

### 3.2 勾配スキャン（N=30・両学習者）

成熟語 MAE footprint（baseline からの上乗せ）とピーク削減のトレードオフ:

| exp | 成熟MAE上乗せ(分散/バースト) | Δピーク(分散/バースト) |
|---|---|---|
| 1.5 | +0.0055 / +0.0029 | −0.8(ns) / −0.4(ns) |
| 2.0 | +0.0047 / +0.0026 | −0.6(ns) / −0.9(ns) |
| **2.5** | **+0.0040 / +0.0024** | **−1.4(2.9σ) / −1.4(2.8σ)** |
| 3.0–4.0 | ≈同等 | 同等〜やや改善 |

**急勾配ほど両軸で良い**（footprint 小 かつ ピーク削減大）。rc=1 の一撃（±50%・exp 非依存）がクリーンな desync を seed し、rc≥2 の後追いノイズ（desync を濁す＋footprint を増やす）が消えるため。**exp=2.5 を採用値**とした。

## 4. 決定打 — throughput 増はラチェット人工物か、本物か

播種で throughput（定着）が両学習者・複数 exp で一貫して有意プラスに出た（バースト +6〜+9.5・3〜4σ）。だが **stage は正解で sticky**（mastered は降格しない）ため、対称ノイズの rc=1 で運良く h0=1.5 を引いた語が mastered 閾値を早く越え、実力以上に居座る**偽 mastered で水増し**された可能性がある（#1 が消した過大評価の再来）。

**アウトカム基準で直接検証**（mastered 数だけでなく「真に覚えてる語数＝期末試験×学習語」「mastered 語の真の保持率」「符号付きバイアス」）:

### 朝バースト 120日（N=30）

| 指標 | OFF → ON | 判定 |
|---|---|---|
| mastered 数 | 280.2 → 298.6（+18.4・5.2σ） | — |
| **真に覚えてる語数** | 301.9 → 316.5（**+14.6・4.9σ**） | mastered と同程度↑＝**本物** |
| mastered 語の真の保持率 | 0.8868 → 0.8866（ns） | 偽 mastered で薄まっていない |
| 符号付きバイアス | +0.0023 → +0.0016（**下降**・6.9σ） | 過信は増えていない |

偽 mastered なら「mastered だけ↑・真に覚↑なし・mastered真p低下・バイアス上方」になるはず。**結果は全部逆 ＝ genuine な学習改善**。60日 +10.5 → 120日 +14.6 と**持続・増大**。

### 分散 180日（N=30）

mastered +0.8・真に覚 +1.6・試験 −0.0012、**全部 ns ＝完全中立**。90日で見えた僅かな試験低下（−0.0027）はノイズで長期消失。

**教訓2（throughput は多指標で）**: mastered 数だけ見ると偽陽性に騙される。「真に覚えてる語数」「mastered の真の保持率」「バイアス方向」を併せて初めて本物と確定できる。

## 5. 結論

| | 朝バースト（過負荷＝位相同期局面） | 分散（余裕あり） |
|---|---|---|
| 真に覚えてる語数 | **+14.6（4.9σ・本物・持続）** | +1.6（ns） |
| 偽 mastered | なし（mastered真p・バイアス健全） | — |
| 下振れ | なし | なし（完全中立） |

**播種ノイズ `base/rc^2.5` は、過負荷の学習者で genuine に +約5% 多くの語を本当に覚えさせ、余裕のある学習者には無害。** 下振れレジームは見つからず＝常時適用で安全（load-gating 不要）。

メカニズム: 容量制約（朝バースト 20枚 vs 需要99）下で、播種が同期した backlog の対称性を破り、一部の語が h を伸ばして渋滞を抜け**本当に mastered になる**（間隔の空いた復習で真の記憶も付く＝spacing effect）。余裕のある学習者には最適化する backlog がないので無害。**勾配が鍵**: 持続ノイズ（√rc）は成熟 h を汚し続けこの便益を相殺した（throughput +1.1 ns）が、播種（rc^2.5）は早期に対称性を破ってから止まるので便益を残しつつ精度を保つ。

当初狙った dueSampling のピーク削減は小さかったが、**本当の payoff は過負荷時の genuine な throughput 改善**だった——朝集中＝位相同期が起きるまさにその局面で。

## 6. 実装（core 採用・2026-06-11）

播種ノイズを **core 実装・既定 true** で採用した。
- `config.js`: `seedNoise: true`・`seedNoiseBase: 0.5`・`seedNoiseExp: 2.5`
- `srs-engine.js` `_applySeedNoise(word)`: 正解時の h 更新後に `h *= 1 + (rand·2−1)·base/rc^exp`（rc=更新後 reviewCount・hMin/hMax クランプ）。適用点は ① intro（rc=1 の一撃）② handwrite 正解 ③ 通常正解（昇格判定の後＝閾値はクリーンな h で判定）。app/sim 両方が `processResponse` 経由。
- **`dueSampling` は既定 false に**: 同じ desync 目的で seedNoise がより強力・outcome 検証済み・両者は冗長で組み合わせ未検証のため。effectiveH/uncertaintyWidth は提案書系譜として残置（`dueSampling=true` で再有効化可能）。
- **検証**: core 実コードパスを burst 120日 N=30 で再検証 → 真に覚えてる語数 +12.5（4.5σ・genuine・mastered真p 不変・バイアス下降）。harness 注入版（+14.6）と同等。
- **標準 3/日 sim baseline は不変**（D90 264・D180 ~472。seedNoise は spread で中立・dueSampling off も throughput 中立）。

### 残課題（任意）
- dueSampling + seedNoise の組み合わせは未検証（必要なら検証して両立可否を判断）。
- 実ドッグフーダー（上級・3/日）が burst 寄りか spread 寄りかで体感便益が変わる。実使用ログで確認の余地。
