# Word Wave — AI 協働による間隔反復アルゴリズムの実証実験

このリポジトリの主眼は、語彙学習アプリという**プロダクトの開発ではない**。その裏で動く独自の間隔反復（SRS）アルゴリズムを、**人間と AI の協働によって設計・校正・実証する**ことが目的である。スワイプ式の学習アプリ（Word Wave）は、そのアルゴリズムを実際に走らせ、シミュレーションと実機ドッグフーディングの両面から検証するための**実験台**として位置づけている。

- **製品表示名**: Word Wave（ユーザー向けの名称）
- **開発コードネーム**: VocabFlow（ファイル名・クラス名・localStorage キー等で使用）
- **アルゴリズム名**: **SRSウェーブ方式（with Ripple Seeding）**

---

## 何を実証しようとしているか

> 既存の SRS（SuperMemo / Anki の SM-2、Duolingo の HLR 等）は「いつ復習すべきか」を点推定でスケジュールする。本実験の問いは、**「記憶の不確実性そのものを最小の機構で行動に変換すると、過負荷の学習者の定着が改善するか」**である。

中心的な主張は次の3点に集約される。

1. **独自性** — 記憶の半減期に「信頼度連動の播種ノイズ」を播くことで、同時に投入された語の**位相同期（同時 due → 同時復習 → 復習の山）を構造的に解す**。コホート同期を崩す目的は Anki の fuzz factor 等と共通だが、予定日に毎回引き直す一過性のジッタではなく、**記憶状態 h に保存型のノイズを加え、その幅を復習回数で急減衰させる**点が異なる（次回間隔だけ見れば h へのノイズと間隔へのノイズは乗法的に等価だが、本方式は摂動を状態に残し以後の成長にも作用させ、ノイズをほぼ導入時に集中させる）。保存型・自己減衰型の最小機構である。
2. **妥当性** — 思いつきではなく、**間隔効果を内蔵した独立な「真の記憶モデル」を持つ仮想学習者**でシミュレーション検証している。アウトカム（実際に覚えているか）を多指標で測り、タイミング系の改善は **N≥20 + 標準誤差**で統計的に判定する規律を課している。
3. **適用可能性** — 中心機構は HLR を持つ任意の SRS に移植できる **rc²（復習回数の二乗）勾配ひとつ**で達成されており、観測が正誤の二値しかない実運用でも成立する。

## アルゴリズムの独自性 — 海のメタファー三層

設計を水文学の三層スケールで呼び分ける。それぞれ独立した時間スケールの「揺らぎ／投入／周期」を担う。

| 層 | 機構 | スケール | 役割 |
|---|---|---|---|
| **Ripple（さざ波）** | 半減期 h に播く信頼度連動ノイズ `base / rc^2.5` | 秒〜時間 | 同じ波で入った語を少しずつ違うタイミングで岸に返す |
| **Wave（波）** | 新語投入のゲート（100語＝1波） | 数日〜週 | 新語を段階供給。復習はプール全体を横断 |
| **Tide（潮）** | コホート全体が満ち引きする周期 | 週〜月 | 位相同期の正体。満ち／引き／凪として可視化 |

独自性の核は **Ripple Seeding**（播種ノイズ）にある。鍵は**勾配の急峻さ**だ。緩い勾配（`√rc`）は成熟語まで複利的にノイズを蓄積し h を汚すが、急勾配（`rc^2.5`）は**導入時（rc=1）の一撃に播種を集中させ、rc≥2 で実質ゼロになる**。「導入時の一回だけ播いて複利を止める」ことで、不確実な新語のコホートだけを解しつつ、観測の積み上がった熟知語の推定は一切汚さない。

さらに2つの校正機構が支える。

- **deltaT 連動ゲイン**: 正解時の h 成長を前回復習からの実経過時間で減衰させる。`ratio = min(1, deltaT / (h × retentionFactor))` で正規化し、**予定どおりの復習は full gain・クラミング（詰め込み）は成長抑制**とする＝間隔反復の本質をモデルに織り込む。
- **不確実性連動スケジューリング**: 観測の少なさ・古さから h 推定の幅を導出し、due 判定にトンプソンサンプリング様のゆらぎを与える（既定では播種ノイズに役割を譲り無効）。

### 正直な性質の明示

本方式は**教科書的ベイジアン SRS ではない**。h は点推定で分布を持たず、正誤は α/β のルールベース乗算（ベイズ更新ではない）、不確実性の幅は事後分散ではなくヒューリスティック導出、rc² 播種は事後分布からのサンプリングではなくトンプソンサンプリングの「形」を借りたものである。正確には **「ベイジアンに着想を得た、不確実性連動の播種ノイズを持つ HLR」** と呼ぶのが誠実だ。価値はベイズの機械の実装ではなく、**不確実性を行動に変換すること**を最小の機構で達成した点にある。

## 妥当性 — 検証フレームワーク

アルゴリズム実験として、主張は再現可能なシミュレーションで裏づける。

- **独立した真の記憶モデルを持つ仮想学習者** (`sim/virtual-learner.js`) — システムが推定する h とは**独立した真の半減期** `trueH` を語ごとに保持し、間隔効果（最適間隔の成功は durable に効く／massed な詰め込みは効かない）を内蔵する。これにより「h は伸びたが実際には定着していない」状況を表現でき、校正機構の真価を測れる。語ごとの個体差（±30%）でシステム非観測の推定誤差も再現する。
- **アウトカムを多指標で測る** — `mastered` カウント単体は「sticky なステージに運良く居座る偽 mastered」に騙される。そこで**真に覚えている語数・mastered 語の真の保持率・符号付きバイアス**を併せて見る。
- **統計的規律** — タイミング系の throughput は run 間分散が真値を上回るため、**N≥20 + 標準誤差**で符号を確定させる（小標本で符号が二転三転した教訓を `seed-noise-findings.md` に記録）。

### 主要な検証結果（要約）

| 機構 | 学習者 | 結果 | 判定 |
|---|---|---|---|
| deltaT 連動ゲイン | 標準（3回/日） | 校正 MAE 0.173 → **0.013**（約 1/13）。バイアス +0.17 → +0.007 | 校正・定着とも改善 |
| Ripple Seeding | 過負荷（朝バースト・N=30） | 真に覚えている語数 **+12.5（4.5σ）**・偽 mastered なし | genuine な +約5% |
| Ripple Seeding | 余裕のある学習者 | 下振れなし・完全中立 | 常時適用で無害 |
| 不確実性 due ゆらぎ | 初学者／既習（N=24） | Δ定着は SE 内（有意差立証できず）。需要 CV のみ再現的に低下 | **効果は立証できなかった**（既定 false） |

最後の行が重要だ。**立証できなかった機構を「効果なし」と正直に記録し既定を off にする**ことも、この実験の成果の一部である。シミュレーションの定着ペース（1000語を 365日内に未到達＝詰め込み不可の honest な値）も、楽観的に膨らませず保守側に倒している。

## 記憶コアは支配的レバーではない — Ebisu 検証の結論

README を強気に書いた直後、本物のベイズ SRS **Ebisu**（fasiha/ebisu）を発見し、「記憶コアを丸ごと置き換えるべきか」を sim で実証した（全文 `memory-core-investigation.md`・13章）。結論は反直感的だった。

- **物差しの転換** — 「予測 p と真の保持率の校正 MAE」は **circular**（真実カーブを固定した瞬間に勝者が決まる）。非 circular な物差しとして **対オラクル％**を導入した。オラクル＝我々と全く同じ全系で、recall 推定だけを学習者の**真のカーブ**に差し替えた仮想の神様。ours/oracle 比は「推定誤差のコスト」だけを測る。
- **真のレバーは記憶コアではなく新語供給構造** — 新語の供給を絞らない健全な構造なら、現実的（べき則）忘却の下で **HLR / Ebisu / DSR のどのコアでもオラクルの 96〜99%** に届く。「全部 Ebisu に置換せよ」は誤りだった。
- **`%oracle` は絶対崩壊を隠す** — 観測成功率で新語を絞る「適応導入」は `%oracle` では near-oracle に見えたが、**製品の実定義（mastered＝ステージ成熟・avgH）で測り直すと、指数則・べき則・ACT-R のどの真実でも greedy に惨敗**した（指数則 novice で 264 → 2）。policy-matched オラクルとの比は、両者とも低いと ~100% に見えてしまう。**絶対アウトカムの併読が必須**という方法論上の教訓。
- **greedy（現行既定）が真実不確実性に頑健** — 真の忘却カーブの形を sim では当てられない以上、どの真実でも崩れない既定を選ぶのが安全。適応導入・新語予約・最低導入フロアはいずれも不採用で確定し、既定は **指数則 HLR + greedy + Ripple Seeding** のまま据え置いた（候補は gated で残置）。
- **唯一の外部審判は実データ** — 「現実の忘却がべき則か」は sim では決着できず、**復習ログから forgetting 曲線の形を測る**ことだけが答えになる。これは今後の課題として開いている（特に復習需要が容量に張り付く初学者層は、上級者のドッグフーディングでは観測できない）。

この章の価値は「Ebisu を採らなかった」ことではなく、**自分の強気な結論を検証台が自己反証できることを示した**点にある。

## AI 協働のかたち

この実験は、設計・実装・検証・データ生成のすべてで AI（Claude）と協働している。

- **アルゴリズム設計の対話** — 海のメタファー三層・播種勾配の選定・「h は seed であって測定値ではない」という再フレーム等は、AI との設計対話から生まれた。
- **コードレビューと反証** — SRS エンジンの構造レビュー（`review.md` の指摘 #1〜#6）を AI が起票し、一件ずつ sim before/after を回して潰した。
- **検証スクリプトの共同設計** — `scripts/verify_*.js` 群。小標本ノイズに二度騙された経緯ごと記録している。
- **教材データ生成** — 1900語の語義・例文・語源解説・トリビア等を Claude API でバッチ生成し、多段の品質監査（事実誤認・論理矛盾・選択肢の同義衝突）を経て確定。

## 適用可能性と限界

**移植性** — 中心機構（Ripple Seeding）は「更新後の h に `base / rc^exp` のノイズを乗せる」一行で、HLR / 半減期型の状態を持つ任意の SRS に組み込める。新たな観測チャネル（反応時間・自己評価など）を要求せず、正誤の二値だけで成立する。

**適用が効くレジーム** — 効果は**過負荷の学習者（多数の語が同時に due になる位相同期局面）**で顕在化する。余裕のある学習者には中立（無害）なので、常時適用が安全。

**限界（正直に）** — deltaT 連動ゲインの校正優位は「真の記憶が間隔則に従う」前提に依存する。模型非依存に主張できるのは「経過時間を無視すると保持率を一貫して過大評価する」までで、優位の大きさは真の間隔則の形に依存する。真正ベイズとの対決（事後分布維持の旨味）は、観測が二値のため薄い見込みであり未実施。

## 実験台としてのアプリ（Word Wave）

アルゴリズムを実際に体験・観測するための学習 UI。

- **6 種類のカード** — Intro（導入）/ Recognition（四択認識）/ Recall（想起）/ Dictation（書き取り）/ Handwrite（手書き介入）/ Passive（語源・コツ・トリビアの読み物）
- **状態推定としての SRS** — `new → recognition → recall → dictation → mastered` をステージ昇格。不正解で降格。「覚えた」という終着点を置かず、mastered も h が閾値を超えた状態にすぎない
- **離脱を設計に織り込む** — 1週間／1ヶ月空いても、戻った瞬間に「今いちばん危ない語」を再構成。ストリークに依存しない
- **可視化** — Wave ヒートマップ（全1900語の記憶強度）／ Word Wave 画面（水深ランプ・Tide の満ち引き・全Wave 制覇ペース）／ Marine Chart 学習プロファイル（誤答の渦・乗り越えた難所・綴りの暗礁）
- **ビルド不要** — Vanilla JavaScript（ES Modules）。フレームワーク・npm 依存・ビルドツールなし

## アーキテクチャ

3 層構成。`core/` は UI に一切依存しない純粋ロジックで、`sim/`（シミュレーター）と `app/`（実 UI）の両方が同じ `core/` を共有する。**SRS ポリシーを core に一元化**し、app/sim が同じ判定経路を通ることで、シミュレーターで検証した挙動が実アプリと乖離しないことを保証する。

```
        ┌─────────┐     ┌─────────┐
        │  sim/   │     │  app/   │
        └────┬────┘     └────┬────┘
             └───────┬───────┘
                ┌────┴────┐
                │  core/  │  ← SRS ロジック（UI 非依存・唯一のポリシー源）
                └─────────┘
```

主なモジュール:

| ファイル | 役割 |
|---|---|
| `core/srs-engine.js` | 半減期 h の更新・deltaT ゲイン・Ripple Seeding・ステージ遷移 |
| `core/wave-manager.js` | Wave の供給ベース解放・卒業判定 |
| `core/feed-generator.js` | 1 セッション分のカード列を貪欲方式で生成 |
| `core/word-data.js` | 全 1900 語の教材データ（Claude API で生成） |
| `sim/virtual-learner.js` | 間隔効果を内蔵した独立な真の記憶モデル |
| `sim/sim-runner.js` | 仮想学習者による長期シミュレーション |
| `app/app.js` | セッション管理・スワイプ・localStorage 永続化 |
