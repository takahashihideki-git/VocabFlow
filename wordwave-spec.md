# Word Wave — UI仕様書

## 概要

Word Wave は VocabFlow の学習進捗を「波」として可視化するインタラクティブ画面。1900語を提示順（ID順）に並べ、各単語の半減期 h に応じた色分けで定着の波が右に広がっていく様子を表現する。

Word Cloud のように単語が読める形で並ぶが、フォントサイズの差ではなくヒートマップ的な色分けが特徴。

## 2. 表示仕様

### 2.1 レイアウト

- 単語は Wave 順（ID順）に左上から右下へ流れるように配置する
- 各単語は inline-block でフローレイアウト（CSS word-wrap）
- 単語間の間隔は最小限（詰め込み配置）
- 画面は縦スクロール可能

### 2.2 ズームレベル

フッタのスライダーでフォントサイズを連続的に変更する。

| ズーム | フォントサイズ目安 | 見え方 | 用途 |
|---|---|---|---|
| 最小（デフォルト） | 6-8px | 単語は判読困難。全体が太いヒートマップに見える | 全体俯瞰。波の進行を確認 |
| 中間 | 12-14px | 単語が読める。スクロールで全体を見渡せる | 特定ウェーブの確認 |
| 最大 | 20-24px | 大きく読みやすい。タップ操作がしやすい | 個別単語の操作 |

```javascript
// ズームスライダー
const ZOOM_MIN = 6;   // px
const ZOOM_MAX = 24;  // px
const ZOOM_DEFAULT = ZOOM_MIN;  // 初期状態は最小
```

### 2.3 カラーマッピング

各単語の背景色（またはテキスト色）を h に応じて設定する。

| h の範囲 | 色 | 状態 |
|---|---|---|
| 未学習 (stage='new') | #E0E0E0 (グレー) | 未着手 |
| h < 1日 | #FF4444 (赤) | 初期段階、不安定 |
| 1日 ≤ h < 3日 | #FF8C00 (オレンジ) | Recognition〜Recall段階 |
| 3日 ≤ h < 7日 | #FFD700 (黄) | Dictation段階 |
| 7日 ≤ h < 14日 | #9ACD32 (黄緑) | 定着進行中 |
| 14日 ≤ h < 30日 | #32CD32 (緑) | ほぼ定着 |
| h ≥ 30日 | #006400 (深緑) | 完全定着 |
| 除外 | #999999 (暗いグレー) + 取り消し線 | 学習対象外 |

テキスト色は背景色とのコントラストを確保する（暗い背景には白文字、明るい背景には黒文字）。

### 2.4 ウェーブ境界の表示

50語ごと（waveSize）に視覚的な区切りを入れる。

- 方法: ウェーブ間に少し広めのマージン、または薄い縦線
- ウェーブ番号ラベル（"W1", "W2", ...）を各ウェーブの先頭に小さく表示
- アクティブウェーブは枠線やバッジで強調

## 3. サマリー統計（ヘッダ部）

Word Wave 画面のヘッダに常時表示する統計情報。

```
Word Wave                                    ✕
学習: 315/1900  定着: 276  Wave 8  avgH: 109日
```

| 項目 | 表示 | 計算 |
|---|---|---|
| 学習語数 | 315/1900 | stage ≠ 'new' の語数 / 全語数 |
| 定着語数 | 276 | h ≥ 14日（masteredThresholdH）の語数 |
| 現在 Wave | Wave 8 | アクティブウェーブの最大番号 |
| 平均 h | 109日 | h > 0 の語の平均半減期 |

## 4. 単語タップ操作

### 4.1 タップ時のポップオーバー

ズームが十分大きい状態（フォントサイズ ≥ 14px 目安）で単語をタップすると、ポップオーバーが表示される。

```
┌─────────────────────────┐
│ abandon                 │
│ /əˈbændən/              │
│ 〜を捨てる、放棄する      │
├─────────────────────────┤
│ Stage:  Dictation       │
│ h:      12.3日          │
│ peakH:  15.1日          │
│ Reviews: 8回 (正解6)     │
├─────────────────────────┤
│ [除外する]               │
└─────────────────────────┘
```

### 4.2 除外操作

「除外する」をタップすると、その単語を学習対象から外す。

- 確認ダイアログ: 「"abandon" を学習対象から除外しますか？（後から戻せます）」
- 除外された語は `excluded = true` フラグが設定される
- 表示: 暗いグレー (#999999) + 取り消し線
- SRSへの影響: フィード生成時に候補プールから除外される。h, stage はそのまま保持
- 復元: 再度タップして「除外を解除」で復元可能

**WordState への追加フィールド:**
```javascript
this.excluded = false;  // 学習対象から除外されているか
```

**フィード生成への影響:**
```javascript
// 候補プール構築時、excluded な語をスキップ
if (w.excluded) continue;
```

## 5. ヘッダバーのミニ表示

メイン学習画面のヘッダに表示するミニ版 Wave Heatmap。

### 5.1 表示

- 1900語を1ピクセル幅（または画面幅 / 1900）の縦棒として横一列に並べる
- 各棒の色は Section 2.3 のカラーマッピングに従う
- 高さ: 8-12px 程度
- タップ可能であることを示すため、薄い枠線を付ける

```
████████████████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░
```

### 5.2 リアルタイム更新

- カード回答ごとに、対応する単語の色が変化する
- ウェーブ解放時に新しい領域が灰色→赤に変化する

## 6. 実装詳細

### 6.1 ファイル配置

```
app/
├── ui-wordwave.js    # Word Wave のロジック・描画
└── (app.html に統合)  # Word Wave のHTML構造
```

### 6.2 描画方式

- **全画面 Word Wave:** DOM ベース（各単語が span 要素）。CSS で色・サイズを制御
- **ヘッダバー:** Canvas ベース（1900ピクセルの描画。DOM では重すぎる）

### 6.3 パフォーマンス考慮

- 1900個の span 要素は DOM として許容範囲（仮想スクロールは不要）
- ズーム変更時は CSS font-size の変更のみ（DOM の再構築不要）
- 色の更新は個別の span の style.backgroundColor を変更（全体再描画不要）

### 6.4 ズームスライダー

```html
<input type="range" min="6" max="24" value="6" id="zoom-slider">
```

```javascript
slider.addEventListener('input', (e) => {
  const fontSize = e.target.value;
  wordwaveContainer.style.fontSize = `${fontSize}px`;
});
```

### 6.5 データ連携

Word Wave は LearnerState の words 配列を直接参照する。

```javascript
function renderWordWave(learnerState) {
  for (const word of learnerState.words) {
    const span = document.getElementById(`ww-${word.wordId}`);
    span.style.backgroundColor = getColorForH(word);
    span.style.textDecoration = word.excluded ? 'line-through' : 'none';
  }
}
```

カード回答後に毎回 `updateWordWaveCell(wordId)` を呼び、該当する1つの span の色だけを更新する。
