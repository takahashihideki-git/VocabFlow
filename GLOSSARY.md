# VocabFlow 用語集（SRS・実験・独自命名）

ドキュメントやコードコメントに出てくる専門用語・略語・独自の造語を平易に解説するための索引です。「急にこの単語を出されても分からない」を減らすのが目的なので、**数式の暗記ではなく「何のために存在するか」**を中心に書いています。

> メンテ方針は末尾の[「この用語集の育て方」](#この用語集の育て方)を参照。新しい略語・造語を導入したら、ここに1行足してから本文で使う運用にします。

---

## まず押さえる5語（これだけで大半が読める）

| 用語 | ひとことで言うと |
|---|---|
| **h（半減期 / half-life）** | その単語を覚えている強さ。「あと何日で記憶が50%まで薄れるか」を日数で表した、SRSの心臓部の状態変数。大きいほど強い。 |
| **p（保持率 / recall probability）** | 「今この瞬間、正しく思い出せる確率」(0〜1)。`p = 2^(−経過日数 / h)` で、時間が経つと下がり、復習すると h が伸びて回復する。 |
| **Wave（波）** | 新しい単語を解禁する単位（100語＝1波）。波が「満ちる」と次の単語群が届く。 |
| **mastered（定着）** | しっかり覚えた状態（h≥14日 かつ Dictation 通過）。アプリの達成度はこの語数で測る。 |
| **sim / 仮想学習者（VirtualLearner）** | 実ユーザーの代わりにコードが回す「シミュレーション上の架空の学習者」。アルゴリズム変更の良し悪しを、本番に出す前にここで測る。 |

---

## 1. 統計・実験で出てくる言葉

検証ログ（`seed-noise-findings.md`・`memory-core-investigation.md` など）で頻出。「この改善は本物か？」を判断するための語彙です。

| 用語 | 読み/正式名 | 意味（このプロジェクトでの使われ方） |
|---|---|---|
| **MAE** | Mean Absolute Error（平均絶対誤差） | 予測した保持率と実際の差の平均。校正の良さを測る指標だが、「真の記憶モデル」の仮定を変えると勝者が入れ替わる**循環的な物差し**なので注意（→絶対値も併読）。 |
| **CV** | Coefficient of Variation（変動係数） | ばらつきの大きさ（標準偏差÷平均）。「同じ日に入れた語が一斉に復習期限を迎える」混雑を測るのに使う。 |
| **SE** | Standard Error（標準誤差） | 平均値そのものの誤差。シミュレーションは毎回ブレるので、効果を主張するには**SE併記が必須**。 |
| **N≥20** | サンプルサイズ | 何回シミュレーションを回したか。タイミング系の効果は N が小さい（例 N=5）と符号すら定まらないため、最低20回が目安。 |
| **バイアス / 分散** | bias / variance | 誤差の2成分。バイアス＝系統的なズレ（過大/過小評価＝**害**）、分散＝単なるばらつき（必ずしも害ではない）。播種ノイズは分散を増やすが害はない、という判断に使う。 |
| **アーティファクト** | artifact | 本当の効果ではなく、測り方の都合で見えただけの偽の結果。例：旧モデル下で出た「+7%」。 |
| **交絡** | こうらく / confounding | 複数の原因が絡んで真因が分からない状態。「ある条件でだけ勝つ」効果は交絡を疑う。 |
| **EWMA** | 指数加重移動平均 | 直近ほど重みを大きくした平均。「最近の正答率」をなめらかに追うのに使う。 |
| **σ（シグマ）** | 標準偏差 / 不確実性 | ばらつきの幅。旧設計で「h推定の自信度」を持たせようとしたが更新則が誤っていて削除された経緯がある。 |

## 2. 記憶モデル・SRSアルゴリズムの名前

「記憶の忘れ方をどう数式で表すか」の流派です。VocabFlow はこれらを**差し替え可能**にして比較実験している（`memoryCore` 切替）。

| 用語 | 正式名/系統 | 意味 |
|---|---|---|
| **HLR** | Half-Life Regression | 本プロジェクトの**既定**。半減期 h で指数的に忘れるモデル（`p=2^(−Δt/h)`）。 |
| **指数則 (exponential)** | — | 忘却が速い形の曲線。HLR の土台。安全側に倒れる代わり「過剰に復習させがち」。 |
| **べき則 (power law)** | — | 指数則より**忘れるのが遅い**形の曲線。FSRS など実データはこちらに近いとされる。「現実はべき則」かどうかが本プロジェクトの大きな賭けの論点。 |
| **FSRS** | Free Spaced Repetition Scheduler | Anki などで使われる現代的SRS。べき則系・難易度/安定度モデル。比較対象・将来の差し替え候補。 |
| **DSR** | Difficulty / Stability / Retrievability | FSRS系の3変数モデル。べき則寄りの「現実的な忘却」を検証するために実装（`memoryCore='dsr'`）。 |
| **Ebisu** | — | ベイズ流の記憶モデル。保持率を確率分布として持ち更新する。比較用に実装（`memoryCore='ebisu'`）。 |
| **間隔効果 (spacing effect)** | — | 復習を**分散**させた方が、詰め込みより記憶に残る現象。仮想学習者がこれを再現するよう作ってある。 |
| **massed（集中）/ cram（詰め込み）** | — | 短時間に繰り返すこと。間隔効果の逆で、h があまり伸びない＝「一夜漬けは身につかない」をモデル化。 |
| **ベイズ / ベイジアン** | Bayesian | 不確実性を確率分布として扱う考え方。本プロジェクトは「ベイズに着想を得た」設計で、厳密なベイズSRSではない。 |
| **トンプソンサンプリング** | Thompson Sampling | 不確実性の幅から値を1つ引いて行動を決める手法。復習期限の同期をばらす案として検討（現在は未採用）。 |

## 3. このプロジェクト独自の命名・造語（海のメタファー）

製品名 **Word Wave** に合わせ、機構を水の三層スケールで呼び分けています。

| 用語 | 意味 |
|---|---|
| **Ripple（さざ波）** | 個々の語の h にまく小さな揺らぎ（＝**播種ノイズ / seedNoise**）。同じ波で入った語の復習タイミングを少しずつズラす。秒〜時間スケール。 |
| **Wave（波）** | 新語を解禁するゲート。100語＝1波。数日〜週スケール。 |
| **Tide（潮）** | 学習者全体が「一斉に新語期（満ち潮）→一斉に復習定着期（引き潮）」になる周期。＝**位相同期**の正体。Word Wave 画面で満ち/引き/凪として可視化。 |
| **Ripple Seeding（播種ノイズ）** | 正解時の h 更新に信頼度連動のノイズを加える機構。`base/rc^2.5` の勾配で「導入直後（rc=1）に一回だけ強くまき、以降ほぼゼロ」になるのが肝。位相同期を散らす。 |
| **SRSウェーブ方式** | HLR＋Wave解放＋貪欲スロット配分＋セッション上限を束ねた本アプリのSRS全体設計の呼称。外向きには「SRSウェーブ方式（with Ripple Seeding）」。 |
| **位相同期 (phase locking)** | 同じ日に入れた語が h・最終復習日・復習回数まで揃い、復習期限が一斉に来て渋滞する現象。Ripple/Tide はこれを和らげるための仕掛け。 |
| **信頼度ゲート** | 復習3回未満（reviewCount<3）の語を Word Wave で別色（泡色）にして「出会ったばかり＝まだ未知数」と表示する仕組み。播種ノイズによる h のゆらぎを、実力差のように見せないため。 |
| **Marine Chart（学習プロファイル）** | Word Wave 右下の FAB から開く全画面の分析ビュー（誤答の渦・乗り越えた難所・綴りの暗礁）。既存データの可視化のみでSRSには触れない。 |
| **綴りの暗礁 (reef)** | 「意味は分かるが綴れない」で停滞している語。プロファイルの一区画＆特訓ドリルの対象。 |
| **対オラクル％** | 「recall推定だけを学習者の真の記憶に置き換えた理想システム（オラクル）」と比べた達成率。MAEの循環性を避けるための非循環な物差し。ただし**両方低いと~100%に見える**ので絶対値の併読が必須。 |
| **CRN** | Common Random Numbers（共通乱数）。比較したい2条件（ON/OFF）で乱数の種を共有し、run間のブレを相殺するペア比較の技法。`core/rng.js`。 |
| **genuine（本物の定着）** | 偽 mastered（運良く高 h に居座っているが実際は覚えていない語）を除いた「期末試験で本当に正解できる語数」。効果が本物か判定する中心指標。 |
| **death spiral / review wall** | 復習債務が雪だるま化して新語が入らなくなる破綻状態（デススパイラル／復習の壁）。適応導入を不採用にした主因。 |
| **死にフィールド / 形骸化** | 保存されるがどこからも使われない状態変数（旧 μ/σ など）。レビューで削除された。 |

## 4. config / コードのパラメータ（`core/config.js`）

「値をいじると挙動が変わるツマミ」。既定値は変更されることがあるので最終確認は `core/config.js` を見てください。

### 記憶の伸び縮み
| パラメータ | 既定 | 役割 |
|---|---|---|
| `h0` | 1.0 | 新語の初期半減期（日）。 |
| `alpha` | 2.0 | 正解時に h を何倍に伸ばすか。 |
| `beta` | 0.3 | 不正解時に h を何倍に縮めるか。 |
| `deltaTGain` | true | 正解時の伸びを「前回からどれだけ時間が空いたか」で割引く（＝間隔効果の実装）。詰め込みでは伸びない。 |
| `retentionFactor` | 0.234 | 「予定どおりの復習間隔」を定義する係数。deltaTGain の正規化基準。 |
| `targetRetention` | 0.85 | 保持率がこの値（85%）まで下がったら復習期限とみなす。 |
| `cardWeight` | 種別ごと | カード難度の重み（recognition 0.8 / recall 1.0 / dictation 1.3 / handwrite 1.6）。難しい形式ほど h が伸びる。 |

### ノイズ・不確実性
| パラメータ | 既定 | 役割 |
|---|---|---|
| `seedNoise` / `seedNoiseBase` / `seedNoiseExp` | true / 0.5 / 2.5 | 播種ノイズ（Ripple）の有無と強さ・勾配。 |
| `dueSampling` | false | 復習期限を不確実性の幅から揺らす機構。seedNoise に役割を譲り既定オフ。 |
| `uncertaintyBase` / `uncertaintyFloor` / `staleGrowth` | 0.5 / 0.05 / 0.05 | h 推定の「自信のなさの幅」を観測回数と放置時間から導く係数群。 |

### ステージ・閾値
| パラメータ | 既定 | 役割 |
|---|---|---|
| `masteredThresholdH` | 14.0 | 「定着」と認める h の下限（日）。 |
| `dictationThresholdH` | 4.0 | Dictation に進ませる h の下限。 |
| `graduationH` / `waveUnlockH` | 8.0 | Wave 卒業判定に使う h の下限。 |
| `handwriteStuckThreshold` | 3 | 同じ段階で何回つまずいたら手書き介入するか。 |

### セッション・供給
| パラメータ | 既定 | 役割 |
|---|---|---|
| `sessionSize` | 20 | 1セッションの最大カード数（認知負荷の主ツマミ）。 |
| `maxNewPerSession` | 5 | 1セッションの最大新語数。 |
| `waveSize` | 100 | 1波あたりの語数。 |
| `adaptiveNew` / `adaptiveNewSignal` | false | 新語枠を負荷で動的調整する案（**不採用・既定オフ**。greedy 維持）。 |
| `successEWMAAlpha` | 0.05 | 直近成功率（EWMA）の更新の速さ。 |
| `memoryCore` | 'hlr' | 使う記憶モデル（'hlr'本番 / 'ebisu' / 'dsr' は検証用）。 |
| `rng` | Math.random | 乱数源。検証では決定論的な種を注入する（→CRN）。 |

### 状態変数（`core/models.js`）
| 名前 | 役割 |
|---|---|
| `peakH` | その語が過去に到達した最大の h（瞬間値でなく履歴最高）。主に表示・sim用。 |
| `reviewCount` (rc) | その語の復習回数。信頼度ゲートや播種ノイズの勾配に使う。 |
| `stage` | 学習段階（new → recognition → recall → dictation → mastered）。 |
| `successRate` | エンジンが見ている直近の実測正答率（EWMA）。推定値でなく実測なので頑健。 |

## 5. 仕組み・パイプラインの概念

| 用語 | 意味 |
|---|---|
| **貪欲方式 (greedy allocation)** | 毎セッションを「緊急→期限→新語」の優先順で埋める方式。復習が多ければ新語は自動で減る＝意図的な負荷調整。`feed-generator.js`。 |
| **Wave 卒業 (graduation)** | ある波の語の90%以上が h≥8 になり、残り新語も尽きたら、その波を「アクティブ」から外すこと。 |
| **スナップショット復元** | Dictation で惜しい誤答時、入力前の状態を保存→いったん不正解登録→直せば保存状態を戻して正解上書き、というUI上の巻き戻し。 |
| **リトライ設計（2026-04-21〜）** | 不正解カードは「降格後の段階」のカードとして再出題し、必ず通常のSRS処理を通す方式。SRSロジックを一本化するための設計。 |
| **真実モデル (trueModel)** | 仮想学習者が「本当はどう忘れるか」の設定。`alpha`(指数・HLR同族) / `dsr`(べき則) / `ebisu`(ベイズ) を切り替えて、各アルゴリズムを公平に比較する。 |
| **slipRate / guessRate** | 観測の揺らぎ。slip＝覚えてるのにミス（タイポ）、guess＝覚えてないのに正解（まぐれ）。 |
| **hFactor（個体差）** | 語ごとの「覚えやすさ」の係数。システムからは見えない誤差源として仕込み、アルゴリズムが「推定の外れ」を直せるかを測る。 |

## この用語集の育て方

- **新しい略語・造語をコードやドキュメントに導入したら、本文で使う前にここへ1行追加する**のを習慣にします（読み手＝未来の自分が迷子にならないため）。
- 既定値（config）が変わったら表の数値も更新。迷ったら正は `core/config.js`。
- 各用語は「数式の正確さ」より「何のために・どこで出るか」を優先。深掘りは出典ファイル（`spec.md` / `review.md` / `seed-noise-findings.md` / `memory-core-investigation.md` / `bayesian-srs-proposal.md`）へ。
- カテゴリは ①統計 ②記憶モデル ③独自命名 ④パラメータ ⑤概念 の5本立て。新語は近いところへ。
